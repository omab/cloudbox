Why LocalGCP

Everything you need for local GCP development

Like LocalStack, but purpose-built for Google Cloud.

🚀

Zero GCP credentials

Develop and test against real-looking GCP APIs without a Google account, billing, or service account keys.

🔌

SDK-compatible

Works with the official google-cloud-* Python SDKs. Point them at LocalGCP and your code runs unchanged.

⚡

Instant startup

All services start in under a second. No Docker images to pull, no network calls, no cold-start delays.

🐍

Pure Python

Built with FastAPI and uvicorn. Easy to inspect, extend, and debug. DuckDB powers BigQuery and Spanner.

🖥️

Admin UI

Built-in web dashboard at localhost:8888 — browse buckets, topics, secrets, tasks, and more in real time.

💾

Optional persistence

State is in-memory by default. Set LOCALGCP_DATA_DIR to persist data across restarts via atomic JSON writes.

Quick Start

Up and running in seconds

Choose Docker for the quickest setup, or run locally with uv.

1

Docker (recommended)

Pull and start all services with a single command:

docker compose up

Admin UI available at http://localhost:8888

2

Local Python

Requires Python 3.12+ and uv.

uv sync
uv run localgcp
3

Install from PyPI

Install directly and run:

pip install localgcp
localgcp
SDK Compatibility

Use the official Google Cloud SDKs

Point the google-cloud-* libraries at LocalGCP — your application code stays identical.

Cloud Storage

from google.cloud import storage

client = storage.Client(
    project="local-project",
    client_options={"api_endpoint": "http://localhost:4443"},
    credentials=AnonymousCredentials(),
)
bucket = client.bucket("my-bucket")

Pub/Sub

# gRPC (standard emulator pattern)
export PUBSUB_EMULATOR_HOST=localhost:8085

# Python
from google.cloud import pubsub_v1
publisher = pubsub_v1.PublisherClient()
topic_path = publisher.topic_path("local-project", "my-topic")
publisher.create_topic(name=topic_path)

BigQuery

from google.cloud import bigquery

client = bigquery.Client(
    project="local-project",
    client_options={"api_endpoint": "http://localhost:9050"},
    credentials=AnonymousCredentials(),
)
result = client.query("SELECT 1 AS n").result()
for row in result:
    print(row.n)

Secret Manager

from google.cloud import secretmanager

client = secretmanager.SecretManagerServiceClient(
    client_options={"api_endpoint": "localhost:8090"},
    credentials=AnonymousCredentials(),
)
secret = client.create_secret(
    request={"parent": "projects/local-project",
              "secret_id": "my-secret", "secret": {}}
)

Pre-configured helpers are available in sdk_compat/clients.py — import and go.

CLI Tools

Drop-in replacements for gcloud and gsutil

Two CLI entry points let you use existing shell scripts without modification.

gsutillocal — Cloud Storage CLI

gsutillocal ls                         # list buckets
gsutillocal ls gs://my-bucket         # list objects
gsutillocal mb gs://my-bucket         # create bucket
gsutillocal cp ./file.txt gs://b/f    # upload
gsutillocal cp gs://b/f ./file.txt    # download
gsutillocal rm gs://b/file.txt        # delete
gsutillocal mv gs://b/old gs://b/new  # move

gcloudlocal — GCP multi-service CLI

gcloudlocal pubsub topics create my-topic
gcloudlocal pubsub subscriptions create my-sub \
    --topic my-topic
gcloudlocal secrets create my-secret
gcloudlocal secrets versions add my-secret \
    --data "s3cr3t"
gcloudlocal tasks queues create my-queue
gcloudlocal scheduler jobs list
